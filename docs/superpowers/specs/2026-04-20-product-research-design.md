# Product Research Feature Design

## Overview

新增 `product-research` 任务类型，根据 JD 输出目标公司/产品的介绍、竞品分析、面试问题预测和准备建议。帮助候选人在面试前快速建立对目标产品的认知。

## 输入

- **JD**（必需）：文件或内联文本
- **profile_store.yaml**（可选）：有则生成个性化面试预测和关联点分析

## 触发条件

- 用户明确要求了解某个公司/产品
- 用户要求做面试前的产品认知准备
- 不触发：用户只要面试题单（mock interview）或简历改写

## 输出

单个 Markdown 文件，保存到 `outputs/姓名_公司_岗位_产品研究_v1.md`。不输出 PDF。

### 文档结构

```markdown
# 产品研究 — {公司} {岗位}

- 候选人: {姓名}
- 目标岗位: {公司} - {岗位}
- 生成时间: {时间}

## 一、产品介绍
- 产品定位与背景
- 核心功能模块（表格）
- 客户群体
- 商业模式
- （如 JD 涉及特定技术方向）该方向的详细能力现状

## 二、竞品分析
- 直接竞品对比表（维度：定位、功能、价格、技术路线、优劣势）
- 该产品的差异化优势

## 三、面试问题预测
- 产品理解类（考察对产品的认知深度）
- 技术方案类（考察 JD 中提到的技术方向）
- 产品设计类（考察产品思维）
- 结合背景类（有 profile_store 时生成）

## 四、你的关联点（有 profile_store 时生成）
- 哪些经历和这个岗位直接相关
- 面试中主动展示的切入点建议

## 五、准备清单
- 必须了解的概念（checkbox list）
- 建议体验的产品/功能
- 建议阅读的材料
```

## 执行流程

1. **解析 JD** — 提取公司名、产品名、岗位职责中的关键技术方向和业务场景
2. **联网调研** — 搜索产品官网、功能介绍、行业评价、竞品信息、公司最新技术布局
3. **生成产品介绍 + 竞品分析**（Section 一、二）
4. **生成面试问题预测**（Section 三）
   - 无 profile_store：仅基于 JD 职责推测通用问题
   - 有 profile_store：额外生成"结合背景类"问题
5. **生成关联点分析 + 准备清单**（Section 四、五）
   - 无 profile_store：跳过 Section 四，Section 五仅输出通用准备建议
   - 有 profile_store：完整输出
6. **保存文件** — `outputs/姓名_公司_岗位_产品研究_v1.md`

## 与现有功能的关系

- 完全独立的任务类型，不依赖也不触发其他任务
- 用户可以先跑 product-research 再跑 mock interview，但两者不强制串联
- 信息来源：AI 知识 + 联网搜索，不要求用户提供额外材料

## 实现方式

- 新建 `skill-pack/PRODUCT_RESEARCH.md` 作为独立 skill 文档
- WORKFLOW.md 新增 `product-research` task type，指向该文档
