# Design: 模拟面试功能（技术面试）

Date: 2026-04-15

## 背景

OfferPilot 已有简历优化、定向改写、JD 匹配、求职信、职位发现等功能。求职瓶颈往往不在简历而在面试。模拟面试是自然的产品延伸——已有 JD 内容和候选人 profile，这就是模拟面试需要的全部输入。

## 范围

- 聚焦技术面试（非行为面试、HR 面）
- 纯 skill-pack 驱动，不新增 Python 脚本
- 分两阶段：题单生成 → 对话模拟

## 工作流

### 阶段一：题单生成

1. 输入：JD 文件 + profile_store.yaml
2. agent 解析 JD 提取技术要求，与 profile 做 gap analysis（复用 JD_MATCHING.md 匹配逻辑）
3. 生成 8-12 道技术面试题，分两类：
   - 强项深挖（约 60%）：围绕候选人已有经历追问细节，标注 `[强项深挖]`
   - 弱点考察（约 40%）：针对 JD 要求但 profile 中薄弱的领域，标注 `[弱点考察]`
4. 每道题附带：难度（基础/中等/进阶）、考察点、参考答案要点
5. 输出 Markdown 保存到 `outputs/`，命名 `姓名_公司_岗位_面试题单_v1.md`

### 题单结构

```markdown
# 技术面试题单 — {公司} {岗位}

- 候选人: {姓名}
- 目标岗位: {公司} - {岗位}
- 生成时间: {时间}
- 题目数量: {N} 题（深挖 {X} + 弱点 {Y}）

## 匹配概览
（简要列出强项和弱点领域）

## 题目

### Q1 [强项深挖] [中等] Android 性能优化
**题目：** ...
**考察点：** ...
**参考答案要点：** ...
```

### 阶段二：对话模拟

**面试官行为规则：**

- 按题单顺序提问，每次只问一题
- 用户回答后追问 1-2 个 follow-up（细节、边界情况、替代方案）
- 每题结束给一句简短反馈，不打断节奏
- 不主动给出参考答案，除非用户明确要求
- 语气专业但不刁难

**流程控制：**

- 用户可说"跳过"进入下一题
- 用户可说"结束"提前终止，直接进入评估
- agent 每 3-4 题主动问一次"继续还是先到这里？"

### 评估报告

对话结束后生成评估报告，保存到 `outputs/`，命名 `姓名_公司_岗位_面试评估_v1.md`。

```markdown
# 模拟面试评估 — {公司} {岗位}

- 候选人: {姓名}
- 面试时间: {时间}
- 完成题数: {N}/{总数}

## 总体评价
（2-3 句话概括）

## 逐题评估

### Q1 Android 性能优化 — ⭐⭐⭐⭐ (4/5)
- **亮点：** ...
- **不足：** ...
- **改进建议：** ...

## 复习清单

| 优先级 | 知识点 | 原因 | 建议学习方式 |
|--------|--------|------|-------------|
| 🔴 高 | ... | ... | ... |
| 🟡 中 | ... | ... | ... |
| 🟢 低 | ... | ... | ... |
```
