# Structured Evaluation

Use this workflow for quantified job fit evaluation (结构化岗位评估).

## Overview

对目标岗位进行 10 维度量化评估，输出 A-F 等级和综合结论。比 JD 匹配度分析更全面，覆盖薪资、成长空间、公司阶段等非技能维度。

## Inputs

Required:
- job description (JD file or inline text)
- profile datastore (`profile_store.yaml`)

## Scoring Dimensions

| # | 维度 | 权重 | 评估要点 |
|---|------|------|---------|
| 1 | 核心技能匹配 | 15% | JD 要求的硬技能与 profile 的匹配程度 |
| 2 | 经历相关性 | 15% | 过往经历与目标岗位职责的重合度 |
| 3 | 级别匹配 | 10% | 工作年限和资历是否匹配岗位级别 |
| 4 | 学历匹配 | 10% | 学历、专业、学校是否满足要求 |
| 5 | 行业契合度 | 10% | 候选人所在行业与目标行业的关联 |
| 6 | 成长空间 | 10% | 岗位和公司能提供的成长机会 |
| 7 | 薪资竞争力 | 10% | 岗位薪资水平是否有吸引力 |
| 8 | 地理位置 | 5% | 工作地点是否在候选人偏好范围内 |
| 9 | 公司阶段/规模 | 10% | 公司发展阶段和规模是否匹配候选人偏好 |
| 10 | 文化契合度 | 5% | 团队文化、工作方式是否适合候选人 |

每个维度打 1-5 分，附一句话理由。

## Grade Mapping

| 总分 | 等级 | 建议 |
|------|------|------|
| 4.5+ | A | 强烈推荐投递 |
| 3.5+ | B | 推荐投递 |
| 2.5+ | C | 可以考虑 |
| 1.5+ | D | 不太推荐 |
| <1.5 | F | 不推荐 |

```markdown
| 维度 | 权重 | 得分 | 理由 |
|------|------|------|------|
| ... | ... | ... | ... |

## 投递建议
...
```

## Batch Evaluation

批量评估多个 JD 时：
1. 逐个读取 JD 内容
2. 对每个 JD 执行上述评估流程
3. 输出汇总排名表，按总分降序
4. 保存到 `outputs/resumes/`

## File Naming

- 单个评估: `outputs/resumes/姓名_公司_岗位_评估_v1.md`
- 批量汇总: `outputs/resumes/姓名_批量评估汇总排名.md`
